## Architecture

### Phased pipeline

The `Makefile` orchestrates phases sequentially (`make all`). Each phase is a standalone `scripts/*.sh` shell script Рђћ all use `#!/bin/zsh` with `set -euo pipefail`. Every script is **idempotent** (safe to re-run).

| Phase | Script | Depends on |
|-------|--------|------------|
| bootstrap | `scripts/bootstrap.sh` | Рђћ |
| brew | `scripts/install_brew_bundle.sh` | bootstrap |
| shell | `scripts/setup_shell.sh` | brew |
| languages | `scripts/setup_languages.sh` | brew, shell |
| git | `scripts/setup_git.sh` | brew |
| containers | `scripts/setup_containers.sh` | brew |
| apps | `scripts/setup_apps.sh` | brew |
| validate | `scripts/validate.sh` | all above |

Scripts that need `brew` include a PATH-detection block probing `/opt/homebrew` and `/usr/local`.

### Brewfile as package manifest

`Brewfile` is the single source of truth for all packages Рђћ CLI tools, casks (GUI apps), fonts. Phases reference this same file; there is no separate package list.

### Dual Git identity

The most complex subsystem (`setup_git.sh`). Design:
- **Global config** (`user.name`/`user.email`) holds the default (corporate) identity.
- **`~/.gitconfig-github`** holds the GitHub identity as a separate file.
- **`init.templateDir`** points to `~/.git-template/hooks/` which contains a `post-checkout` hook. Every `git clone` fires this hook on first checkout: if the remote origin is `github.com`, it adds `include.path ~/.gitconfig-github` as a local config entry.
- **Two SSH keys**: `id_ed25519` (default) and `id_ed25519_github` (for `github.com` in `~/.ssh/config`).
- Supports both interactive prompts and environment variable overrides (CI mode).

### Language runtimes

Python (pyenv), Ruby (rbenv), Node (volta), Go (manual install), Rust (rustup). Default versions defined as variables at the top of `setup_languages.sh`. All overridable via environment variables.

### Alternative deploy paths

- **chezmoi** (`chezmoi/` directory): Templates (`dot_<filename>.tmpl`) for dotfiles. Templates use Go-style `{{ .git.fieldName }}` syntax. Some templates are auto-generated by `setup_languages.sh` and `setup_git.sh`.
- **Ansible** (`ansible/`): Single playbook that replicates the shell pipeline declaratively. Useful for multi-machine setups.

### Variables

Key defaults in `setup_languages.sh`:
- `PYTHON_VERSION=3.11.6`
- `RUBY_VERSION=3.2.2`
- `NODE_VERSION=20`

Override any by setting the environment variable before running the script (e.g. `NODE_VERSION=22 make languages`).

Git/GitHub identity variables for non-interactive mode: `GIT_NAME`, `GIT_EMAIL`, `GIT_GITHUB_EMAIL`, `GIT_GH_USER`.

### How to add a package

1. Add the cask/formula/tap to `Brewfile` under the appropriate section.
2. If a new setup script is needed, place it in `scripts/` and add a `Makefile` target.
3. Add a corresponding check in `scripts/validate.sh`.
4. If managed by chezmoi, add a template in `chezmoi/`.
